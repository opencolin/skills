# Event Page Generator

Turn a rough event concept into a finished, publish-ready event page. The output should be good enough to paste directly into Luma, Eventbrite, or a landing page with minimal edits.

## Operating principles

The reader is a busy developer or builder deciding in ten seconds whether this event is worth their evening. Lead with the value to them, not the logistics. Be specific and concrete. Avoid hype words and filler. Write in a confident, warm, operator voice. No emojis. No em dashes.

If key details are missing (date, location, audience, format), make reasonable assumptions and clearly mark them as `[assumption: ...]` so the user can correct them. Do not stall the draft waiting for perfect inputs.

## Required output structure

Always produce the page in this exact order so it is copy-paste ready:

### 1. Title options
Give 3 title options, ranked. The top option should be the recommended one. Each title is short, specific, and signals the value or the vibe. Avoid generic names like "Tech Meetup."

### 2. One-line hook
A single sentence, under 20 words, that makes the target attendee want to read more.

### 3. The description body
Three short paragraphs:
- What this is and who it is for
- What attendees will get, see, or do (be concrete: demos, speakers, hands-on, networking)
- Why now, or why this room is worth being in

### 4. Agenda
A simple time-blocked agenda. Keep it realistic. Include arrival/networking, the main content, and a wind-down. Use placeholder times if none were given and mark them as assumptions.

### 5. Host and sponsor block
A 2 to 3 sentence host bio written in first person if the host is the user, plus a line for any sponsors or partners with space to add logos.

### 6. Logistics block
Date, time, location, capacity, cost (default free unless told otherwise), and any access notes (badge, RSVP required, building entry). Format as a clean scannable list.

### 7. Call to action
One clear CTA line for the RSVP button and one sentence beneath it that adds urgency without being pushy (limited capacity, prior events sold out, etc., only if true or marked as an assumption).

### 8. Reminder sequence
Three short messages the host can schedule:
- T-minus 1 week (build anticipation, share one detail)
- T-minus 1 day (logistics, what to bring, arrival info)
- Day-of (final nudge, door info)

## Example

**Input:** "AI builders happy hour at Frontier Tower, want devs to come hang and see a few demos"

**Output (title options):**
1. Build & Brews: AI Builders Happy Hour at Frontier Tower (recommended)
2. Demos and Drinks: An Evening for AI Builders
3. Frontier Tower AI Mixer

Then proceed through every section above.

## Quality bar

Before returning, check: would a developer who has never heard of this event understand it, want to come, and know exactly how to RSVP, all from this page alone? If not, tighten it.

End the output with a soft, single line the user can keep or delete:
