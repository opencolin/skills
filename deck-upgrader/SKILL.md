# Deck Upgrader

Take a flat deck or outline and turn it into a spec for an interactive web slideshow that is clearer, more engaging, and on-brand. The goal is a deck that holds attention and is built for the web, not a static export.

## Operating principles

A strong deck has one idea per slide, a narrative arc, and visuals that carry meaning rather than decorate. Cut filler, sharpen the throughline, and let interactivity earn its place (it should aid understanding, not distract). Theme everything to the brand if one is given. No em dashes in slide copy.

## Inputs and setup

Accept bullets, an outline, exported slides, or notes, plus any brand reference (colors, fonts, tone, logo). Identify the deck's single goal and audience before restructuring. If the narrative is weak, fix the arc first; design cannot rescue a deck with no story.

## Required output structure

### 1. Narrative arc
The reordered throughline in a few lines: the setup, the turn, the payoff, the ask. This is the spine every slide hangs on.

### 2. Slide-by-slide spec
For each slide: the single message, a tight headline, the body content reduced to essentials, a layout suggestion (e.g. full-bleed visual, split, data focus), and any interactive element (build animation, hover reveal, embedded demo, live chart) where it genuinely helps. Mark where a visual or chart should replace text.

### 3. Brand and visual system
Color usage, type treatment, spacing, and motion guidance themed to the brand. Keep it consistent and restrained.

### 4. Speaker notes
A short note per slide: what the presenter says and the transition into the next slide.

## Quality bar

Before returning, check: does each slide carry one idea, does the arc build to the ask, and does every interactive element aid understanding rather than show off? If a slide is a wall of bullets, redesign it.
